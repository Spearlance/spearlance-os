# Phase 1: P0 Bug Fixes Implementation Plan


**Goal:** Fix four critical bugs: restore broken toast notifications (ALL toasts silently failing), fix `.single()` crashes on empty queries, fix hardcoded 2025 dates, and convert Dashboard from waterfall fetching to useQuery.

**Architecture:** The toast system is the highest priority — NEITHER the radix-ui `<Toaster />` nor the sonner `<Sonner />` component is mounted in the component tree. All 581 toast calls across 148 files are silently swallowed. We'll restore sonner as the single toast system (simpler API, already partially adopted), mount `<Sonner />` in App.tsx, and migrate all 110 `useToast` files to sonner. For `.single()`, we'll convert 59 unsafe query-based calls to `.maybeSingle()` while keeping 28 post-insert `.single()` calls safe as-is.

**Tech Stack:** React 18, Supabase JS v2, TanStack Query 5, sonner, TypeScript

**Migration pattern:**

Each file follows this transformation:

1. Remove: `import { useToast } from "@/hooks/use-toast";`
2. Add: `import { toast } from "sonner";`
3. Remove: `const { toast } = useToast();` from the component body
4. Transform each `toast({...})` call:

| Old (useToast) | New (sonner) |
|----------------|--------------|
| `toast({ title: "Done" })` | `toast.success("Done")` |
| `toast({ title: "Done", description: "Details" })` | `toast.success("Done", { description: "Details" })` |
| `toast({ title: "Error", variant: "destructive" })` | `toast.error("Error")` |
| `toast({ title: "Error", description: "Details", variant: "destructive" })` | `toast.error("Error", { description: "Details" })` |
| `toast({ title: "Info" })` (no variant, contextually informational) | `toast.info("Info")` |

## Task 5: Fix unsafe .single() → .maybeSingle() (query-based calls only)

**Files:**
- Modify: 28 files containing unsafe `.single()` calls (those NOT after insert/upsert)

**Context:** Supabase `.single()` throws a `PGRST116` error when the query returns 0 rows. This crashes the UI. `.maybeSingle()` returns `{ data: null }` instead — safe for optional lookups.

**Safe to keep as `.single()`:** Calls chained after `.insert()`, `.upsert()`, or `.update()` with `.select()` — these always return exactly one row. These are in 28 files and should NOT be changed.

**Unsafe — must change to `.maybeSingle()`:** Calls chained after `.select().eq()` or similar query patterns where the row might not exist. These are in the remaining 28 files (59 individual calls).

For each file listed above, change `.single()` to `.maybeSingle()` on query-based calls ONLY. Do NOT change calls that follow `.insert()` or `.upsert()`.

After changing, verify the consuming code handles `null` data. Most already have null checks like `if (data)` or `if (error)`. Where the code assumes data exists (e.g., `data.id`), add a null guard:

```tsx
// Before (crashes if no row)
const { data } = await supabase.from('profiles').select('*').eq('id', userId).single();
setName(data.name);

// After (safe)
const { data } = await supabase.from('profiles').select('*').eq('id', userId).maybeSingle();
if (data) setName(data.name);
```

## Task 7: Convert Dashboard from useEffect waterfall to useQuery

**Files:**
- Modify: `src/pages/Dashboard.tsx`

**Context:** Dashboard currently uses `useEffect` + `setState` to make 6 sequential Supabase queries. This creates a waterfall (each query waits for the previous), prevents caching, and creates race conditions on client switch.

**Step 1: Refactor Dashboard.tsx**

Replace the monolithic `loadDashboardData` function and its `useEffect` with individual `useQuery` hooks:

```tsx
// Replace the stats state + useEffect + loadDashboardData with:

const { data: taskCounts = { to_do: 0, in_progress: 0 }, isLoading: tasksLoading } = useQuery({
  queryKey: ['dashboard-tasks', selectedClient?.id],
  queryFn: async () => {
    const { data: tasks } = await supabase
      .from('tasks')
      .select('status')
      .eq('client_id', selectedClient!.id);
    return {
      to_do: tasks?.filter(t => t.status === 'to_do').length || 0,
      in_progress: tasks?.filter(t => t.status === 'in_progress').length || 0,
    };
  },
  enabled: !!selectedClient,
});

const { data: formSubmissions, isLoading: submissionsLoading } = useQuery({
  queryKey: ['dashboard-form-submissions', selectedClient?.id],
  queryFn: async () => {
    // ... move the form submissions logic here
    // Return the formSubmissions object
  },
  enabled: !!selectedClient,
});

const { data: recentMeetings = [], isLoading: meetingsLoading } = useQuery({
  queryKey: ['dashboard-meetings', selectedClient?.id],
  queryFn: async () => {
    const { data } = await supabase
      .from('meetings')
      .select('*')
      .eq('client_id', selectedClient!.id)
      .order('date_time', { ascending: false })
      .limit(3);
    return data || [];
  },
  enabled: !!selectedClient,
});

const { data: nextMeeting } = useQuery({
  queryKey: ['dashboard-next-meeting', selectedClient?.id],
  queryFn: async () => {
    const { data } = await supabase
      .from('meetings')
      .select('*')
      .eq('client_id', selectedClient!.id)
      .gte('date_time', new Date().toISOString())
      .order('date_time', { ascending: true })
      .limit(1)
      .maybeSingle();
    return data;
  },
  enabled: !!selectedClient,
});

const { data: marketingTools = [] } = useQuery({
  queryKey: ['dashboard-marketing-tools', selectedClient?.id],
  queryFn: async () => {
    const { data } = await supabase
      .from('marketing_tools')
      .select('id, name, url, logo_url, category')
      .eq('client_id', selectedClient!.id)
      .order('name')
      .limit(8);
    return data || [];
  },
  enabled: !!selectedClient,
});

const dataLoading = tasksLoading || submissionsLoading || meetingsLoading;
```

Remove:
- `const [stats, setStats] = useState<DashboardStats>(...)`
- `const [dataLoading, setDataLoading] = useState(true)`
- The `loadDashboardData` function
- The `useEffect` that calls `loadDashboardData`

Keep:
- `actionPlan` state + `loadDailyActionPlan` (edge function call — harder to convert to useQuery and already works)
- All the rendering logic — just update references from `stats.taskCounts` to `taskCounts`, `stats.formSubmissions` to `formSubmissions`, etc.

**Step 2: Verify**

Run: `npx vitest run && npx vite build 2>&1 | tail -5`
Expected: All tests pass, build succeeds
